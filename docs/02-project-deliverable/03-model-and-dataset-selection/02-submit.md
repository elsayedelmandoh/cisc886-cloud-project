# what to submit

## Section 3 - Model & Dataset Selection (3 marks)

### Model Documentation

1. **Model details table** including:
   - Model name
   - Number of parameters
   - Source link (Unsloth, Hugging Face, or Ollama)
   - Model license

2. **Justification paragraph** explaining:
   - Why this model is appropriate for your chosen task
   - Domain fit (how the pre-training data aligns with your use case)
   - Hardware requirements (why it fits your available compute)

### Dataset Documentation

3. **Dataset details table** including:
   - Dataset name
   - Source link (Hugging Face or other)
   - Dataset license
   - Total number of samples

4. **Train/Validation/Test split** showing:
   - Split counts and percentages
   - Split strategy used

5. **Data leakage avoidance** explaining:
   - How splits were generated to prevent leakage
   - Any deduplication strategy

6. **Sample data** - at least one verbatim example from the dataset

7. **Summary statistics** such as:
   - Token length distribution
   - Language/label distribution

> What to submit: A written report section. No code is required for this section.
